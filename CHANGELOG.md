# 更新日志

## v26.8.1（2026-07-15）

### 新增

- `maibindfish` / `mai绑定水鱼` 支持无参数交互式绑定。
- Bot 会提供水鱼查分器入口，并引导用户在“编辑个人资料”中生成和复制
  `Import-Token`。
- Token 格式错误时继续等待，最多允许输入 3 次；支持发送“取消”退出。
- `今日舞萌` 根据每日人品值发放 BREAK：四舍五入至十位后除以 10，
  每天只可领取一次。
- 自动识别消息图片中的二维码；有效舞萌二维码会进入账号更新流程，普通图片静默忽略。
- 直接发送 SGWCMAID、官方链接或二维码图片时，无账号记录会自动验真绑定/认领。

### 优化

- 用户提交的水鱼 Token 会优先撤回；撤回失败时明确提醒用户手动撤回。
- 绑定成功只显示脱敏 Token，并附带可供管理员追踪的 `REF_ID`。
- 水鱼绑定提示、重试与结果引用触发消息，减少多人同时操作时的歧义。
- 连续签到恢复原奖励曲线，第 5 天以后继续线性增长，不再设置奖励上限。
- 直发舞萌凭据会在绑定后同步 PC，并按水鱼 Token / 落雪 OAuth 状态自动上传；
  PC 服务未配置时仍可完成绑定与上传。
- 机台操作改为全局串行：PC 同步完成后等待 3 秒，再依次执行水鱼与落雪上传；
  不同用户共用同一 keychip 时也不会并发登录互相挤掉。
- 落雪 OAuth 上传直接读取机台全量成绩；Access Token 被拒绝时会强制刷新并重试一次，
  刷新响应未轮换 Refresh Token 时仍可继续使用旧 Token。

### 修复

- 修复发票成功后被错误报告为 `FinishedException()` 的问题。
- 一并修复账号、地图、队列、落雪 B50 等流程中同类的成功结果误报。
- 修复 `mymai` 丢失响应外层 `BanState` 而显示“封禁状态：未知”的问题。
- 修复 team 模式票券查询误传 `userId` 导致 `returnCode=102` / 无法获取；
  现已按 maibot 使用 `qrcode + keychip`，并兼容 `userCharge` 包装与免费票券。

> 从旧版本升级不会修改或清空已有账号、BREAK、签到、Token 或成绩数据。
